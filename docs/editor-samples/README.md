# Editor Highlighting Samples

Open these files in the editor to sanity-check syntax and semantic highlighting after changes.

- `python-semantic-highlighting.py`: decorators, imports, type aliases, multi-line bindings, `lambda`, comprehensions, `global` / `nonlocal`, `except*`
- `typescript-semantic-highlighting.ts`: interfaces, enums, readonly fields, generics, destructuring, default-library calls
- `react-semantic-highlighting.tsx`: typed props, hooks, JSX tags, callbacks, readonly values
- `vue-semantic-highlighting.vue`: `<script setup lang="ts">`, refs, computed values, typed props, template bindings

These samples are intended for manual regression checks and editor QA, not runtime execution.
